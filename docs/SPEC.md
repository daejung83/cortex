# Cortex Brain Format Spec v0.1

## Overview

The Cortex Brain Format is an open, human-readable specification for persistent AI memory. It uses plain markdown files organized into a defined directory structure.

Goals:
- Human-readable and human-editable
- Version-controllable (git)
- Tool-agnostic (works with any AI via MCP)
- Hierarchical context (hot / warm / cold)

---

## Directory Structure

```
<brain-root>/
├── short-term/
│   ├── 2026-04-01.md
│   ├── 2026-04-02.md
│   └── ...
├── long-term/
│   ├── projects.md
│   ├── decisions.md
│   ├── people.md
│   └── <any-topic>.md
├── active-context.md
└── always-on.md
```

---

## File Types

### `short-term/YYYY-MM-DD.md`
**Purpose:** Raw daily session notes. Auto-generated by the curation agent.
**Retention:** 30 days by default, then archived or distilled.
**Format:**
```markdown
# Brain — 2026-04-01

## 09:15
Worked on Lotlytics ZIP calculator widget...

## 14:30
Decision: use Railway for backend deployment because...
```

### `long-term/<topic>.md`
**Purpose:** Curated, durable memory on a topic. Updated by the curation agent after distillation.
**Standard topics:** `projects`, `decisions`, `people`
**Custom topics:** anything (e.g. `recipes`, `workout-plan`, `reading-list`)

### `active-context.md`
**Purpose:** Hot context — distilled from last 48hrs of short-term notes. Auto-rebuilt every 6 hours.
**Token target:** ≤40 lines (~1,000 tokens). Loaded on every AI query.
**Do not edit manually** — overwritten on rebuild.

### `always-on.md`
**Purpose:** Permanent, static context. About you, your stack, your current focus.
**Token target:** ≤20 lines. Always loaded.
**Edit freely** — not auto-rebuilt.

---

## Context Hierarchy

| Layer | File | When loaded | Who writes |
|-------|------|-------------|-----------|
| Hot | `active-context.md` | Every query | Auto (curation agent) |
| Always | `always-on.md` | Every query | Human |
| On-demand | `long-term/*.md` | When AI needs it | Auto + Human |
| Archive | `short-term/*.md` | On search hit | Auto |

---

## MCP Tools

The Cortex MCP server exposes these tools to AI assistants:

| Tool | Description |
|------|-------------|
| `get_context` | Returns active-context + always-on |
| `search_brain` | Full-text search across all files |
| `get_projects` | Returns long-term/projects.md |
| `get_decisions` | Returns long-term/decisions.md |
| `get_long_term(topic)` | Returns any long-term file |
| `log_note(content)` | Appends to today's short-term file |

## Versioning

This is spec v0.1. Breaking changes will increment the major version.
Backward-compatible additions increment the minor version.
